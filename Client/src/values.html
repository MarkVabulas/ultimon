Items of note:

  • The data in this chart will refresh from scratch any time a new value comes in that we haven't seen yet.
  • The "values" shown are only the values of the first iteration, so we aren't constantly having to try to update them (I'm lazy at writing this page)
  • The name that the "id" will look for on the stats page is the first one under root, for example "FGPU1", etc.
  • The "value" field is the value as it will be displayed on the final sensor page. In some cases, or with custom visualizations, we can have factors or special renderers, so these values will be abstracted.
